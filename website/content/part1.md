## 第一部分：Vibe Coding 的技术原理

#### 核心目标：理解"为什么AI能帮我写代

**1. 大语言模型（LLM）基础**

要想理解ai为什么能够帮我们写代码，首先我们需要理解当前的AI编程的大脑--LLM的核心机制，再理解它是如何应用到编程领域的。

LLM的全称是Large Language Model，即大语言模型。2022年底，ChatGPT横空出世，将LLM这一概念带入大众视野。其实LLM的工作原理并不复杂，用一句话就可以概括：一个“词语接龙”游戏。

具体来说，模型在海量的文本数据（书籍、网页、论文、代码）上进行训练。训练任务极其简单：给模型一段文字的前面部分，请你预测下一个词是什么。

* 例如，给模型看：“今天天气真”，它需要预测下一个词最可能是“好”、“热”、“冷”还是“糟糕

这种预测机制我们可以将其简理解成模型求参的过程，经过数十上百亿次的更新参数之后，LLM逐渐学会了人类语言的语法、事实知识、逻辑关系。

需要注意的是，LLM对于自然语言的预测并非是一字一词地预测，而是一个“token”一个“token”的预测。我们可以将token理解为LLM阅读和理解文字时使用的最小单位。我们可以举几个几个直观例子：(此处可以插入token划分插件)

* 苹果 → 1 个 token
* 人工智能 → 常被拆成「人工 / 智能」→ 2 个 token
* hello → 1 个 token
* hello world → 2 个 token

“token”的概念在LLM中是非常重要的，因为LLM是有token上限的，如果在我们与LLM交互的过程中，LLM突然忘记了我们之前与它说过的话，那么很可能的原因就是上下文的token到达了LLM的上限，或者说，LLM的“脑容量”不够了。token的限制正是当前vibe coding性能受限的重要原因之一。当然，token重要的原因还有一个：它决定了我们调用LLM的花费。在vibe coding的过程当中，你使用的token越多，我们的花费就越贵。不过不必担心，这也是本教程的目标之一：让我们学会如何用最少的token数，实现我们想要的功能效果。

那么在理解了LLM的原理之后，我们便很容易理解为什么LLM能够帮助我们编写代码了。对于LLM来说，代码和自然语言是没有本质去别的，都有严格的语法规则（关键词、括号、缩进），都有特定的语义（函数名、变量名），也都有特定的逻辑（循环、条件判断）。也正是因此，用来处理自然语言的大模型，稍加改造（其实只要训练数据中包含大量代码），就能处理代码。


**2.智能体（agent）助力**

有了LLM作为大脑还远远不够，有过AI编程经历的同学们可能对此深有体会。如果我们仅仅使用聊天式的机器人来进行交互编程，我们需要详细上传需求和代码，LLM仅仅能够通过当前对话和历史消息了解我们的需求，并且需要手动复制粘贴，回到编辑器进行测试和运行。这样的交互虽然能够让我们简化一些编写代码的过程，但是仍然不够智能。

为了解决这个问题，vibe_coding 利用了agent（智能体）的架构。我们可以将agent简单地理解成“LLM+tools”.

举个例子

* LLM本身只进行代码语言的处理和生成，相当于我们的“大脑”，只能思考而不能行动。
* tools根据LLM的指令调用相关的工具阅读、修改代码，执行指令，相当于我们的“四肢”，需要收到明确指令后进行调用。

在agent的助力下，我们的vibe coding 工具变得全能，它能够智能在我们写代码时补全或者内联建议，可以打开授权的文件、阅读代码，根据我们的要求修改代码。我们不再需要复制上传代码，vibe coding工具可以自动将代码写入文件并且进行运行，甚至能够自动下载代码执行需要的依赖。总而言之，agent的助力让我们的vibe coding工具变得更加全能，成为我们真正的代码助手。

在理解了vibe coding的基础技术原理之后，我们就可以开始下一步了。接下来，我们将了解当前比较主流的几大vibe coding 工具，正式开始我们的vibe coding之旅。
