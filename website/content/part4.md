## 第四部分：实战：从零到一的完整项目

### 核心目标：串联前三部分内容，展示完整流程

#### 1. 项目选择建议

**理想项目特征**
- 复杂度适中（1-2周可完成）
- 涉及多种技术（前端+后端+数据库）
- 能体现多工具协作
- 有实际应用价值

**推荐项目示例**

**项目1：个人知识管理系统**
- 技术栈：React + Node.js + MongoDB
- 功能：笔记、分类、标签、搜索
- 挑战：全文搜索、实时编辑、版本控制

**项目2：任务管理应用**
- 技术栈：Vue.js + Python + PostgreSQL
- 功能：任务创建、分配、跟踪、提醒
- 挑战：权限管理、通知系统、数据统计

**项目3：在线代码编辑器**
- 技术栈：Next.js + Python + Redis
- 功能：代码编辑、运行、分享、协作
- 挑战：代码执行、沙箱环境、WebSocket通信

#### 2. 分阶段演示

##### 阶段1：用ChatGPT规划技术栈和数据库设计

**提示词**
```
我要开发一个任务管理应用，类似Trello的简化版。

请帮我：
1. 选择合适的技术栈
2. 设计数据库结构
3. 规划API端点
4. 估算开发时间

功能需求：
- 项目管理（创建、编辑、删除）
- 任务卡片（创建、移动、编辑）
- 成员管理（邀请、权限）
- 标签和分类

约束条件：
- 支持实时更新
- 需要用户认证
- 响应式设计
- 易于部署

请提供详细的技术方案和设计文档。
```

**预期输出**
- 技术栈推荐（含理由）
- 数据库ER图
- API端点列表
- 开发计划时间表

##### 阶段2：用Cursor/Copilot编写核心功能

**步骤1：创建项目结构**
```
# 在Cursor中
mkdir task-manager
cd task-manager
npm init -y
```

**步骤2：使用Copilot生成代码**
```
# 输入注释，让Copilot生成代码
// 创建Express服务器
const express = require('express');
const app = express();

// Copilot会自动补全中间件配置
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// [继续让Copilot生成路由和控制器]
```

**步骤3：使用Cursor的Chat功能**
```
@Cursor 请帮我创建一个用户认证中间件
要求：
1. 验证JWT token
2. 从token中提取用户信息
3. 处理token过期情况
```

##### 阶段3：用AI生成测试用例

**单元测试**
```
请为以下函数生成完整的单元测试：

{代码}

测试框架：Jest
要求：
1. 测试正常情况
2. 测试边界条件
3. 测试异常情况
4. 使用mock处理外部依赖
```

**集成测试**
```
请为API端点生成集成测试：

API端点：POST /api/tasks
功能：创建新任务

测试场景：
1. 成功创建任务
2. 缺少必填字段
3. 无效的token
4. 数据库连接失败
```

##### 阶段4：遇到bug时的多轮调试过程

**场景：任务无法保存到数据库**

**第1轮：描述问题**
```
我遇到了一个问题：

错误信息：TypeError: Cannot read property 'id' of undefined
出错位置：Task.create()

相关代码：
{代码片段}

请帮我分析可能的原因。
```

**第2轮：提供更多信息**
```
根据你的建议，我检查了数据模型：

Mongoose Schema:
{Schema定义}

请求数据：
{请求body}

请告诉我问题在哪里。
```

**第3轮：验证修复**
```
我按照你的建议修改了代码：

{修改后的代码}

现在错误变成了：
{新的错误信息}

这是进步还是新问题？接下来该怎么做？
```

##### 阶段5：代码优化与重构（让AI提出改进建议）

**代码审查**
```
请审查以下代码并提供改进建议：

{完整代码或代码片段}

请从以下方面分析：
1. 代码质量和可读性
2. 性能优化空间
3. 安全性问题
4. 最佳实践建议
5. 重构建议
```

**性能优化**
```
我的应用响应时间太慢，请帮我优化：

当前性能：
- API平均响应时间：500ms
- 数据库查询：200ms
- 前端渲染：300ms

请提供：
1. 性能瓶颈分析
2. 优化建议
3. 实施步骤
4. 预期改进效果
```

**重构建议**
```
应用功能已经完成，但代码质量需要提升。

请帮我：
1. 识别代码异味
2. 提出重构方案
3. 优先级排序
4. 提供重构示例

重构目标：
- 提高可维护性
- 减少技术债务
- 提升代码复用
```
