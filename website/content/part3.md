## 第三部分：提示词工程与 Vibe Coding 方法论

### 核心目标：建立一套高效的AI编程工作流

#### 1. 面向代码的提示词设计

##### 清晰表达需求的公式（角色+任务+约束+示例）

**基本模板**
```
【角色】你是一位资深的{语言/领域}开发专家

【任务】请帮我完成以下任务：
{具体任务描述}

【约束条件】
1. {约束1}
2. {约束2}
3. {约束3}

【参考示例】
{示例代码或期望输出}
```

**实战示例**
```
【角色】你是一位资深的Python后端开发专家，擅长FastAPI框架

【任务】请设计一个用户认证系统，包括：
1. 用户注册接口
2. 用户登录接口（使用JWT）
3. 密码加密存储
4. 基础的用户信息管理

【约束条件】
1. 使用FastAPI框架
2. 密码使用bcrypt加密
3. JWT有效期24小时
4. 包含输入数据验证
5. 遵循RESTful设计原则
6. 添加完整的类型注解

【参考示例】
期望的API端点格式：
POST /api/auth/register
POST /api/auth/login
GET /api/auth/me
PUT /api/auth/me
```

##### 代码生成的经典提示模式

**模式1：增量式提示**
```
第1轮：创建一个基础的用户类
第2轮：为用户类添加密码验证方法
第3轮：添加密码重置功能
第4轮：优化错误处理
```

**模式2：对比式提示**
```
请用以下两种方式实现快速排序：
1. 递归方式
2. 迭代方式

请对比两种实现的优缺点，包括：
- 时间复杂度
- 空间复杂度
- 实际性能
- 代码可读性
```

**模式3：场景式提示**
```
场景：电商平台需要处理大量订单

任务：设计一个订单处理系统
- 需要支持高并发
- 需要保证数据一致性
- 需要处理订单状态变更
- 需要支持订单查询

请提供：
1. 系统架构设计
2. 核心代码实现
3. 数据库设计
4. 性能优化建议
```

##### 多语言混合项目的提示技巧

**技巧1：明确语言边界**
```
项目结构：
- frontend/ (React + TypeScript)
- backend/ (Python + FastAPI)
- ml-model/ (Python + PyTorch)

请明确每个组件使用的语言和框架
```

**技巧2：关注接口定义**
```
请先定义前后端的API接口规范：
1. 请求格式
2. 响应格式
3. 错误处理
4. 认证方式
```

**技巧3：统一数据结构**
```
确保前后端使用一致的数据结构：
- User对象
- Product对象
- Order对象

请提供TypeScript和Python的类型定义
```

#### 2. Vibe Coding 的核心工作流

##### Step 1 需求拆解：如何将想法拆解成AI可理解的模块

**拆解原则**
1. **单一职责**：每个模块只做一件事
2. **高内聚低耦合**：模块间依赖最小化
3. **可测试性**：每个模块可独立测试
4. **可复用性**：通用功能抽象为独立模块

**拆解方法**
```
原始需求：开发一个在线笔记应用

拆解为：
1. 用户系统
   - 注册/登录
   - 个人信息管理
   - 权限控制

2. 笔记管理
   - 创建笔记
   - 编辑笔记
   - 删除笔记
   - 搜索笔记

3. 分类系统
   - 创建分类
   - 笔记分类
   - 分类浏览

4. 协作功能
   - 分享笔记
   - 评论功能
   - 版本控制

5. 数据存储
   - 数据库设计
   - 缓存策略
   - 备份机制
```

##### Step 2 原型构建：快速生成可运行的基础版本

**MVP优先策略**
1. **核心功能优先**：先实现最重要的20%功能
2. **简化实现**：使用最简单的方式实现
3. **快速验证**：尽早获得用户反馈
4. **迭代优化**：根据反馈逐步完善

**示例：笔记应用MVP**
```
最小可行产品：
- ✅ 用户注册/登录
- ✅ 创建/编辑笔记
- ✅ 保存到数据库
- ❌ 暂不做：搜索、分享、协作
```

**提示词示例**
```
请帮我创建一个笔记应用的最小可行版本：

技术栈：
- 前端：React + TypeScript
- 后端：Express.js + Node.js
- 数据库：MongoDB

功能需求：
1. 用户注册/登录（使用JWT）
2. 创建笔记（标题+内容）
3. 编辑笔记
4. 查看笔记列表
5. 删除笔记

请提供：
1. 项目结构
2. 核心代码实现
3. API端点定义
4. 数据模型设计
```

##### Step 3 迭代优化：通过多轮对话逐步完善

**迭代策略**
```
第1轮：基础功能实现
- 实现核心CRUD
- 基础UI界面

第2轮：功能完善
- 添加输入验证
- 改进错误处理
- 优化用户体验

第3轮：性能优化
- 添加缓存
- 优化查询
- 减少请求次数

第4轮：功能增强
- 添加搜索
- 实现分类
- 支持标签
```

**对话技巧**
- 每轮专注于一个改进方向
- 基于上一轮结果继续优化
- 保持代码上下文一致性
- 记录每次迭代的改动

##### Step 4 问题诊断：遇到bug时的调试技巧

**系统化调试流程**

**1. 问题定位**
```
请帮我分析以下问题：

错误现象：
- {具体描述错误现象}

错误信息：
- {完整错误堆栈}

相关代码：
- {提供出错的代码}

期望行为：
- {描述期望的正确行为}
```

**2. 问题分析**
```
基于上述错误，请：
1. 分析可能的原因
2. 提供诊断步骤
3. 给出修复方案
4. 说明如何预防类似问题
```

**3. 验证修复**
```
修复后请：
1. 解释修复原理
2. 提供测试用例
3. 说明可能的副作用
4. 建议进一步优化
```

##### Step 5 整合与重构：将AI生成的代码融入项目

**整合检查清单**
- [ ] 代码风格一致
- [ ] 命名规范统一
- [ ] 错误处理完善
- [ ] 添加必要注释
- [ ] 编写单元测试
- [ ] 更新文档
- [ ] 性能评估
- [ ] 安全审查

**重构提示词**
```
请帮我重构以下代码：

当前代码：
{代码片段}

重构目标：
1. 提高可读性
2. 减少重复代码
3. 改善性能
4. 增强可维护性

重构要求：
1. 保持功能不变
2. 添加类型注解
3. 添加文档注释
4. 提供重构说明
```

#### 3. 进阶技巧

##### 用代码解释代码（让AI理解现有代码后再修改）

**技巧**
```
请先理解以下代码：

{代码片段}

然后：
1. 解释代码的功能
2. 分析代码的优缺点
3. 提出改进建议
4. 给出重构版本
```

##### 上下文管理（如何在长对话中保持一致性）

**方法1：定期总结**
```
让我们总结一下当前的进展：

已完成：
1. {已完成的工作}

当前状态：
2. {当前代码状态}

下一步：
3. {接下来的计划}
```

**方法2：维护状态文档**
```
项目状态：

技术栈：
- {技术栈列表}

文件结构：
- {目录结构}

关键决策：
- {重要设计决策}

待办事项：
- {待完成的功能}
```

**方法3：版本控制**
- 每次重大改动后提交代码
- 使用清晰的commit message
- 创建分支进行实验性改动

##### 让AI生成注释和文档

**代码注释**
```
请为以下代码添加详细注释：

{代码}

要求：
1. 每个函数添加docstring
2. 复杂逻辑添加行内注释
3. 说明参数和返回值
4. 指出潜在问题
```

**API文档**
```
请基于以下代码生成API文档：

{代码}

文档格式：
- Markdown
- 包含示例
- 说明参数
- 说明返回值
- 列出错误码
```

##### 提示词模板库的建立

**模板分类**
```
1. 基础功能模板
   - CRUD操作
   - 认证授权
   - 数据验证

2. 算法模板
   - 排序算法
   - 搜索算法
   - 动态规划

3. 架构模板
   - MVC架构
   - 微服务架构
   - 事件驱动架构

4. 测试模板
   - 单元测试
   - 集成测试
   - E2E测试
```

**模板示例**
```
# CRUD操作模板

角色：资深{语言}开发工程师

任务：创建{实体}的CRUD操作

技术栈：
- 语言：{语言}
- 框架：{框架}
- 数据库：{数据库}

功能需求：
1. 创建{实体}
2. 查询{实体}（支持分页）
3. 更新{实体}
4. 删除{实体}

约束条件：
1. 使用{ORM/查询构建器}
2. 添加输入验证
3. 处理异常情况
4. 添加日志记录

请提供：
1. 数据模型定义
2. API端点实现
3. 输入验证规则
4. 错误处理逻辑
```
